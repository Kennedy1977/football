# v0.0.63 - Add manager avatar sprite selection and profile updates

Date: 2026-03-11
Type: feat
Scope: profile

## Summary

- Added manager profile-picture selection using a new manager sprite sheet, persisted avatar data through API/DB, and surfaced the selected portrait across onboarding, profile, home, and top navigation.

## Added

- New manager avatar sprite asset: `apps/web/public/assets/managers/manager-sprite.webp`.
- Reusable avatar UI components:
  - `ManagerAvatar`
  - `ManagerAvatarPicker`
- New API endpoint: `PUT /api/onboarding/manager/avatar` for profile-picture updates after onboarding.

## Changed

- Shared API contracts now include typed manager avatar payloads and manager summary avatar data.
- Dashboard summary responses now always include normalized manager avatar data.
- Onboarding manager creation now stores `avatar_json` + `avatar_frame` with validated sprite-frame data.
- `AppFrame`, `Home`, `Start`, and `Profile` screens now display/use manager portraits instead of initials-only/profile icon placeholders.

## Fixed

- Added robust avatar normalization to prevent malformed legacy avatar JSON from breaking profile rendering.

## Removed

- None.

## Validation

- [x] npm run typecheck
- [x] npm run build
- [ ] Manual QA completed on affected screens/routes.

## Risks / Follow-up

- Profile picture is now selected from a fixed 25-frame sprite sheet (`manager-v1`); future sprite sheets will require extending accepted `spriteSheet` values and picker UX.
