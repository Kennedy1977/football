# v0.0.58 - Enforce commit protocol and runtime version file loading

Date: 2026-03-11
Type: build
Scope: workflow

## Summary

- Enforced the new commit protocol through git hooks and fixed runtime version visibility by generating and loading a dedicated `version.json` file at app startup.

## Added

- Commit protocol hooks:
  - `.githooks/pre-commit`
  - `.githooks/commit-msg`
- Protocol validation scripts:
  - `scripts/protocol-check-staged.mjs`
  - `scripts/protocol-check-commit-msg.mjs`
  - `scripts/setup-git-protocol.mjs`
- New npm scripts:
  - `protocol:install`
  - `protocol:check:staged`
  - `protocol:check:commit-msg`
  - `protocol:prepare`
- Runtime version artifact:
  - `apps/web/public/version.json`

## Changed

- Bumped app version to `0.0.58`.
- `scripts/generate-build-meta.mjs` now:
  - uses `package.json` as the version source of truth
  - writes both `build-meta.ts` and `public/version.json`
- `BuildFooter` now fetches `/version.json` on load (with build-meta fallback).
- `notes:new` now auto-updates patch-note feeds in:
  - `CHANGELOG.md`
  - `docs/implementation-plan.md`
- Updated protocol and patch-note documentation:
  - `docs/git-commit-and-push-format.md`
  - `docs/patch-notes/README.md`
  - `README.md`

## Fixed

- Resolved stale displayed version issue caused by outdated build metadata/version source precedence.

## Removed

- None.

## Risks / Follow-up

- Protocol hooks are local repo config; run `npm run protocol:install` per environment/clone.
- Hook enforcement is strict by design and can block commits until all required files are staged.
