# v0.0.78 - fix shop chest build typing and add unfinished chest notifications

Date: 2026-03-12
Type: fix
Scope: shop

## Summary

- Fixed the Shop compile error caused by chest ribbon type narrowing.
- Added unfinished chest reward recovery in Shop so users can continue unresolved reward decisions.
- Linked notifications directly into Shop chest contexts (`chests` vs `pending`).

## Added

- New endpoint `GET /api/shop/packs/pending` for unresolved chest rewards.
- New client hook `useGetPendingPackRewardsQuery`.
- New `Unfinished Chests` section on `/shop` that shows unresolved rewards across sessions.

## Changed

- Notification wording moved fully to chest terminology.
- Notification links now deep-link into Shop contexts:
  - `/shop?tab=chests`
  - `/shop?tab=pending`
- Shop focus labels now use chest naming (`Player Chest`, `EXP Chest`, `Coin Chest`).

## Fixed

- TypeScript build failure on `app/shop/page.tsx` (`base[tier].ribbon` union property access).
- Users can now resume unfinished chest decisions after navigation/refresh.
- `/shop` build error from `useSearchParams` static prerender constraint (replaced with client-side URL parsing).

- None.
